# GamBallRun

## プログラムの説明

Unityで制作した、ステージを左右に傾けてボールをゴールまで運ぶ3Dアクションゲームです。ステージを傾けてボールの速度を大きくしないとステージを上ることができませんが、速度を大きくしすぎたりステージを傾けすぎたりするとステージから落ちてしまいます。プレイヤーはボールを直接操作せず、制限時間以内にボールをゴールに運びます。クリア時間によってランクが決まり、最高ランクを目指すためにプレイします。

## 操作方法

### タイトル画面
* Space: ゲーム開始
* Escape: 終了

### ゲームプレイ中
* A / D または ← / → : ステージを左右に傾ける
* Space: リプレイ
* Escape: タイトル画面に戻る

## ルール
* 左右にステージを傾けて、制限時間以内にゴールを目指します。
* ボールがステージから落下した場合はスタート地点に戻ります。
* 制限時間になると、ゲームは終了し、タイトル画面に戻るかリプレイするかを選択することができます。
* 制限時間以内にゴールすると、クリアタイムに応じてランクが表示され、タイトル画面に戻るかリプレイするかを選択することができます。
* プレイ中はリプレイできます。また、タイトルに戻ることもできます。

## 実装した機能、工夫した点
* ステージ傾斜によるボール操作: ボールを直接操作するのではなく、ステージを操作することで間接的にボールを操作します。ここに求める操作と実際の挙動に誤差が生まれ、おもしろさを感じることができると考えます。
* タイトル画面: タイトル画面では操作確認画面に遷移することができるようにしています。これにより初めて遊ぶユーザーや操作方法に不安のあるユーザーも問題なく遊ぶことができます。
* ゲーム中UIの実装: ゲーム中にも操作方法を表示しています。また、制限時間の表示、リプレイ、タイトル画面ボタンを実装し、ゲームプレイが便利になる機能を追加しています。
* ゴールタイムによるランク設定、表示: 一度も落とすことなくゴールできるようにするより、制限時間以内に何度も挑戦し、クリアをめざすことにしました。これは、一度も落とすことなくクリアすることを目的とするゲームはクリアしたら目的が果たされ、またプレイしようとならないと考えたためです。クリアタイムによりランク設定をすることで、クリアできてもより高いランクを目指そうと何度もプレイしたくなるようになると考え、そのように実装しました。
* Space / Escapeキーによるショートカット操作: ショートカットキーによりゲーム開始やリプレイ、ゲーム終了、タイトル画面への遷移をすることができるように工夫しました。これにより、ゲームプレイに慣れたユーザーはマウスを使うことなくゲームプレイすることができます。

## 開発中に生じた問題と解決法
### ステージ操作の方向性の調整
当初は左右だけでなく前後にもステージを操作できるように作成していましたが、操作の自由度が高いことにより、ゲームとして何をするのがよいか分かりづらくなりました。そのため、後方にはデフォルトで少し傾け、左右に傾ける操作をすることにプレイヤーが集中し、単純な操作でゲームを楽しむことができるようにしました。
### 難易度の調整
ステージを傾けるという単純な操作であるため、難易度の調整に何度も試行錯誤しました。プレイヤーが数回調整するだけでは、たまたまでもクリアできないようにした上で、次はこうすればよいだろう、と考察をすることができ、クリアを目標に何度も挑戦することができる難易度に調整できるように、自分で調整するだけでなく、友人に遊んでもらい、アドバイスをもらいました。
### Unityの仕組みによる問題
自分はUnityでの開発経験がほぼない状態で本作を制作したため、Unityの機能をおおまかに学習したうえで課題にぶつかるたびに調べることで解決しました。具体的にはボタンが表示されているのに押せない問題はHierarchyの順番を変えることで解決し、ゴール判定が機能しない問題はオブジェクトのTag名を修正することで解決しました。これらの問題を解決する際に、生成AIを用いました。
### UIの調整
最低限の機能を搭載したものを作成し、実際にプレイする中でUIの問題を解決しました。こういったUIがあるとわかりやすい、便利だといったことを考えたり、発売されているゲームをプレイすることでUIのヒントを得ました。また、高ランクを目指すことを目標としているが、プレイ中にリプレイできるようにするのはゲーム性に影響を与える可能性があると考えました。ただ、現在は試行錯誤しやすさを優先し、即時リプレイできるようにしました。よりゲーム性を増すには、スコア減少、ランク評価へのペナルティなど、コストを設けるようにしたいと考えています。

## AI利用箇所
ChatGPTを補助的に使用しました。
* UnityやC#の実装方針の相談
* UnityのUIに関する相談
* エラーが発生した場合の原因の相談、改善策の提示
* タイトル画像、操作方法画面に使用する画像の生成
生成AIから得た情報やコード案はそのまま使用するのではなく、自身で内容を確認し、理解した上でUnity上で実装しました。動作確認、ステージ設計、UI配置、マテリアル設定、難易度調整、最終的な仕様の判断、細かい挙動の調整はすべて自身で行いました。
